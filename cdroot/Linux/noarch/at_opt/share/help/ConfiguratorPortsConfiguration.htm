MFP Ports Configuration

MFP Ports Configuration shows list of available MFP ports, permits to check status of each port and to release a port that remains locked after process owned the port is terminated abnormally.

Ports list

The icon view shows the available MFP ports. Each port has an icon reflecting its type (parallel or USB). Each MFP port also has a text label denoting the name of the corresponding Linux virtual device (such as /dev/mfp0, /dev/mfp1). These device names are used as URI of the installed printers.

Selected port

Shows detailed information about the selected port. This information includes the type of the port (parallel or USB), the name of the attached device and the name of the application using the port at the moment. If no application has claimed the port, status of the port is reported as "Port is unused".

Refresh

Refreshes the entire ports list. Information about the current port status and attached devices is retrieved.

Release port

Releases the selected port. Application use the port loses port connection and, therefore, terminates current printer job or scan session. This command should be applied in case the port is still locked by abnormally terminated application only.
